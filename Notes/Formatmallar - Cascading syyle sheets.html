Formatmallar -
Cascading Style Sheets

En av de viktigaste nyheterna i HTML 4.0 var stödet för formatmallar (style sheets på engelska). HTML utvecklasdes ursprungligen för vetenskapliga rapporter, dokumentation till program och liknande, där innehållet i dokumenten var det viktiga, inte hur de såg ut på skärmen eller på papper.

När man sedan började använda webben till en massa annat ville många få bättre kontroll över hur deras sidor skulle se ut på skärmen. Man hittade på en massa olika trick för att göra layout med tabeller och bilder och Netscape och Microsoft tävlade om att hitta på egna tillägg till HTML så att det skulle gå att få dokumenten finast just på deras webbläsare. Ett tag såg det ganska dystert ut för HTML, eftersom allt fler dokument bara fungerade med en viss webbläsare och ibland bara med en viss version av en webbläsare. HTML-dokument blev också allt svårare att läsa och uppdatera "för hand", eftersom dokumentens innehåll doldes i labyrinter av kryptiska taggar.

Formatmallar löser det här problemet på ett bra sätt. Man skiljer återigen på dokumentens innehåll och presentation genom att införa ett nytt språk för presentationsinformation. Dessutom kan man med formatmallar ange hur dokument ska formateras på ett mycket noggrannare sätt än tidigare. Man kan ha en och samma formatmall för en hel webbplats, och man kan ha flera olika mallar för ett och samma dokument, så att man till exempel får olika utseende på skärmen och på utskrifter.

Observera att det finns webbläsare som inte bryr sig om formatinformation till exempel vissa webbläsare på handdatoter och mobiltelefoner. Det är därför en bra idé att se till att dina dokument ser någorlunda bra ut även i äldre webbläsare. I praktiken är det här sällan något problem, eftersom mallarna bara innehåller just formatinformation. Om mallen till exempel anger att en viss rubriktyp ska visas med fetstil och två centermeters indrag så blir ju inte dokumentet obegripligt bara för att rubriken ser ut som vanligt.

CSS

Det vanligaste språket man använder för att skriva formatmallar kallas Cascading Style Sheets, ofta förkortat till CSS. Det finns inget i HTML som tvingar en att använda just CSS, men eftersom det är det i särklass mest stödda språket kommer jag att använda det här.

CSS nivå 1 fastslogs i slutet av år 1996. Nästa version, CSS nivå 2, kom 1998 och tillförde stöd för olika mediumtyper, nedladdningsbara teckensnitt, utplacering av element och tabeller.

CSS version 2.1 är den version vi använder i den här boken. Det är en förhållandevis liten uppdatering av version 2, med en del rättningar och några nya funktioner som redan stöds av de flesta webbläsarna, CSS 2.1 har i skrivande stund varit en Candidate Recommendation i nästan ett år, vilket innebär att små ändringar fortfarande kan komma att göras i delar som bara stöds av ett fåtal webbläsare, men specifikationen som helhet är stabil och alla delar som fungerar i många webbläsare kommer att finnas kvar oförändrade.

Arbetet med CSS 3 har pågått i många år, men arbetsgruppen valde att först fokusera på att göra klart CSS 2.1 och CSS 3 vilket är långt ifrån klart. Det finns också ett förslag till en minimal version av CSS 2 anpassad för mobiltelefoner och andra små enheter som kallas CSS Mobile Profile, men specifikationen har efter mer än åtta år ännu inte nått fram ens till Candidate Recommendation.

En av anledningarna till att det tar så lång tid att få fram nya standarder är att man numera kräver att det ska finnas två kompletta implementationer av alla detaljer i en specifikation innan den blir en färdig Recommendation. Man tar också fram testviter så att de som utvecklar webbläsare själva lätt ska kunna kontrollera att de uppfyller standarden. Ytterligare en anledning är att standarden är så omfattande. PDF-versionen av specifikationen till CSS 2.1 är mer än 400 sidor lång.

Deklarationer, väljare och egenskaper

En mall som anger att alla rubriker av typer h1 ska vara gröna ser ut så här:

h1 {color: green }

I den här mallen kallas h1 för väljare (selector på engelska), color kallas egenskap (property) och green är egenskapens värde (value). "color: green" kallas tillsammans för en deklaration (declaration). Klamrarna gör det lätt att se var deklarationen för en väljare börjar och slutar. Det finns fler än hundra olika egenskaper som man kan styra med CSS. Här är några exempel, och jag kommer att visa fler senare i kapitlet:

font-size
- textens storlek (teckengrad)
font-family
- textens teckensnitt
background-color
- bakgrundsfärg. Man kan alltså ha olika bakgrundsfärg i olika delar av dokumentet.
letter-spacing
avståndet mellan bokstäverna i texten
border-style
ram runt texten. Värdet kan till exempel vara none (ingen ram), dotted (sträckad ram) eller solid (heldragen linje).

Man kan ha flera väljare och deklarationer i samma mall. Väljarna skiljs åt med komma och deklarationerna med semikolon. Följande mall anger till exempel att både ordnade och oordnade listor skall visas med teckensittet Garamond på gul bakgrund:

ul, ol{
font-family: garamond;
background-color: yellow;
}

Vi kommer att gå igenom mer om CSS efter hand, men det du har sett här räcker så länge. Jag antar att du vill lära dig att använda mallar i HTML-dokument så fort som möjligt.

Mallar i text

Det finns flera olika sätt att lägga in formatmallar i HTML. Vi ska först gå igenom hur man gör en mall för enstaka stycken och sedan hur man gör mallar för hela dokument och mallar i egna dokument. När man gör större webbplatser använder man nästa alltid mallar i fristående dokument, eftersom samma malldokument då kan användas från alla HTML-dokument på webbplatsen.

Attribute style

Antag att du vill att ett visst stycke ska vara inramat. Det går att åstakomma genom att ge taggen <p> attributet style. Attributets värde är en mall utan väljare. Detta exempel visar hur man får en dubbel ram runt ett stycke: Exempel

De enda taggar som inte kan ha attributet style är <base />, <basefont />, <head>, <html>, <meta>, <script>, <style> och <title>. Det kanske är synd att man inte kan ange ett format för dokumenttiteln, men titeln används ju ofta som namn på det fönster som dokumentet visas i och de flesta operativsystem tvingar alla tilläpningar att ha samma teckensnitt i fönstrens namnrader.

Fördefinierade mallar

Det är bra att kunna byta teckensnitt eller bakgrundsfärg på en rubrik eller några ord, men om man till exempel vill att alla stycken i ett långt dokument ska ha ett visst teckensnitt blir det ganska många style-attribut att skriva. Det är inte speciellt lätthanterligt om man vill göra ändringar, och dokumentet blir både längre och svårare att läsa. De här problemen slipper man om man använder fördefinierade mallar.

En fördefinierad mall är helt enkelt en mall som är definierad i förväg, antingen i början av HTML-dokumentet eller i en separat fil. En av de viktigaste poängen med formatmallar är ju att man kan skilja på innehåll och presentation, och när man definierar mallarna i förväg blir HTML-koden ren och enkel. Då är det lätt att förstå och ändra i HTML-koden utan att man råkar förstöra utformningen.

Taggen <style>

Förväxla inte attributet style med taggen <style>. Attributet används till att ändra formatet på ett enda ställe i ett dokument, men taggen <style> kan påverka formatet på alla förekomster av en viss typ av text, till exempel alla stycken eller alla numrerade listor i hela dokumentet.

Taggen <style> används precis som attributet med samma namn för att ange en formatmall, men man använder <style> i sidhuvudet. Den angivna mallen gäller sedan i resten av dokumentet. Taggen <style> kan ha tre attribut: type, media och title.

type

Attributet type används för att ange vilket språk som används i formatmallen. För mallar skrivna i CSS ska attributet ha värdet text/css. CSS är det förvalda mallspråket, så type behövs inte om du inte använder flera mallspråk. Mer om detta i slutet av kapitlet.

media

Med hjälp av media kan man lägga in olika formatmallar för olika typer av presentationer, så man kan göra en mall för visning på skärmen och en annan för utskrifter. Det här är de värden som attributet kan ha:

screen
avsett för visning på vanlig bildskärm.
print
avsett för utskrifter
handheld
avsett för handdatorer. De har ofta mycket liten skärm och begränsad bandbredd.
tty
avsett för utskrift på enheter med fast tecken-bredd, till exempel gamla teletype-terminaler eller små handdatorer.
tv
avsett för visning på tv. Tv-apparater har låg upplösning jämfört med bildskärmar och man kan inte räkna med att användarna kan rulla texten.
projection
avsett för projicerade presentationer, där en sida eller bild i taget visas.
aural
avsett för talsyntes. Det är inte bara synskadade som har nytta av syntetiskt tal; nu när kvaliteten på talet börjar bli uthärdligt kan det ibland vara ganska skönt att låta datorn läsa upp långa texter medan man lutar sig tillbaka i stolen och vilar ögonen.
braille
avsett för punktskrift på speciella punktdisplayer som synskadade kan använda för att tillgodogöra sig informationen på webben
embossed
nästan som braille men avsett för skrivare som skriver punktskrift för synskadade.
all
lämpligt för alla enheter.

De förvalda värdet är screen enligt HTML-standarden, men de flesta webbläsare använder mallar utan media-attribut både till skärmen och till utskrifter, så i praktiken är media="all" det förvalda. Man kan ange flera medier genom att skilja dem åt med kommatecken.
