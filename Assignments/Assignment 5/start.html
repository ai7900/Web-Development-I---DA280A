Skillnaden mellan inline CSS och extern stilmall

Det finns flera olika sätt att styla ett HTML dokument. Två av sätten är inline style och extern stilmall.

Inline style är när man sätter ett style element p åvarje html-tagg man vill styla. Inom de style element skriver man den CSS kod man vill styla HTML koden med. Man kan skriva hur många attribut man vill inom style element, det finns ingen begränsning.

Fördelen med inline style är att man kan vara väldigt specifik för vilket HTML-tagg man vill styla. Den är bra att använda när man ska styla med Javascript som ofta använder inline style.
Nackdelen med inline style är dock att det lätt kan bli rörigt att ha CSS kod och HTML kod tillsammans i samma dokument, det kan även bli krångligt att göra ändringar i efterhand. Äldre webbläsare kan till och med tolka CSS koden som html kod och därav visa upp det direkt på webbsidan.

Med external stylesheet så har man ett separat CSS dokument där man stylar webbsidan. Detta innebär att alla html dokument som har ett link-element i head taggen som är länkat till CSS-dokumentet får den stilen som är defininerad i CSS-dokumentet. Alla HTML-sidor får nu samma styling som använder sig av samma dokument.

Fördelen med en extern stilmall är att man samlar all styling på samma ställe. Detta gör att det blir enklare att återanvända CSS klasser till alla html dokument som är kopplade till CSS-dokumentet. Koden i html-dokumentet blir även renare och mer lättläst genom att separera HTML- och CSS-kod. Nackdelen med en extern stilmall är att om man stylar alla HTML sidor i samma CSS dokument kan det dokumentet bli väldigt långt och rörigt.
